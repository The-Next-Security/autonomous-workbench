# HEARTBEAT.md — Frontend Developer

Cada ciclo (30 min):

1. ¿PRs del frontend con CI rojo? Si sí, diagnosticar.
2. ¿Historias del sprint asignadas a mí sin avance > 1 día? Reportar
   blockers a Roy.
3. ¿Bundle size actual vs baseline? Si incrementó > 5%, investigar.
4. ¿Hay feedback de QA pendiente en un PR propio? Atender.
5. ¿Cambios en contratos API del backend que afecten código FE? Coordinar.

Si nada aplica, HEARTBEAT_OK.
